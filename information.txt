# Project Overview

TNEA Guider is a full-stack college recommendation system that predicts Dream, Moderate, and Safe colleges for a student based on cutoff, category, branch, budget, and optional district.

The system includes:
- Spring Boot backend API
- MySQL schema and sample data scripts
- Bootstrap-based responsive frontend
- End-to-end scoring and classification logic
- Validation and error handling for robust API behavior

# Database Schema

Database name: `tnea_guider`

Tables:

1. `colleges`
   - `id` (PK, auto increment)
   - `name`
   - `branch`
   - `category`
   - `district`
   - `cutoff`
   - `fees`
   - `placement_rate`
   - `tier`

2. `students`
   - `id` (PK, auto increment)
   - `cutoff`
   - `category`
   - `branch`
   - `budget`
   - `district`
   - `created_at`

3. `recommendations`
   - `id` (PK, auto increment)
   - `student_id` (FK -> students.id)
   - `college_id` (FK -> colleges.id)
   - `final_score`
   - `classification`
   - `created_at`

4. `weights`
   - `id` (PK, auto increment)
   - `weight_key` (unique)
   - `weight_value`

# API Details

Base URL: `http://localhost:8080`

Endpoint: `POST /api/recommend`

Additional endpoint: `GET /api/compare?ids=1,2,3`

Request:

```json
{
  "cutoff": 187.5,
  "category": "BC",
  "branch": "CSE",
  "budget": 100000,
  "district": "Chennai"
}
```

Response:

```json
{
  "dream": [],
  "moderate": [],
  "safe": []
}
```

Validation:
- `cutoff` must be integer or `.5` only
- Category required
- Branch optional
- Budget optional (minimum 50000 when provided)
- District optional

Error Handling:
- `400` for invalid input/body
- `404` when no colleges match
- `503` for database access issues
- `500` for unexpected errors
- `400` for invalid compare id query values

# Backend Logic

Filtering pipeline:
1. Fetch all colleges
2. Always filter by `category`
3. Filter by `branch` only when provided
4. Filter by `budget` only when provided
5. Filter by `district` only when provided

Scoring:

- `cutoff_score = student_cutoff / college_cutoff`
- `tier_weight:`
  - Tier 1 = 1.0
  - Tier 2 = 0.7
  - Tier 3 = 0.5
- `fee_score = 1 - (college_fees / max_fees)`
- `final_score = 0.4*cutoff_score + 0.2*(placement_rate/100) + 0.2*tier_weight + 0.2*fee_score`

Dynamic preference weighting:
- Balanced (default): cutoff 0.4, placement 0.2, tier 0.2, fees 0.2
- High Placement: cutoff 0.3, placement 0.4, tier 0.2, fees 0.1
- Low Fees: cutoff 0.3, placement 0.15, tier 0.15, fees 0.4
- Top Tier: cutoff 0.3, placement 0.15, tier 0.4, fees 0.15
- Weights are auto-normalized to preserve sum = 1 and prevent negatives

Classification:
- Dream: `college_cutoff >= student_cutoff`
- Moderate: within `±5`
- Safe: lower than that range

Persistence:
- Incoming student snapshot stored in `students`
- Each returned recommendation stored in `recommendations`

# How to Run

## 1) Database Setup

Run MySQL scripts in order:
1. `database/schema.sql`
2. `database/sample_data.sql`

Optional Docker startup:

```bash
cd database
docker compose up -d
```

## 2) Backend Setup

Update password in:
- `backend/src/main/resources/application.properties`

Run backend:

```bash
cd backend
./mvnw spring-boot
```

Windows:

```bat
cd backend
mvnw.cmd spring-boot
```

Server starts at: `http://localhost:8080`

## 3) Frontend Setup

Serve `frontend` folder using Live Server on port `5500`.

Open:
- `http://localhost:5500`

## 4) Backend Tests

```bash
cd backend
./mvnw test
```
